## 1 需求分析

### 1.1 项目背景

在学校教学管理中，学生成绩信息需要统一录入、查询和统计。传统纸质或 Excel 管理方式效率低、易出错，不便于多用户协作。因此开发一个基于 Web 的学生成绩管理系统，实现信息化管理。

### 1.2 功能需求

| 编号 | 功能 | 说明 |
|------|------|------|
| F01 | 用户登录 | 管理员登录系统 |
| F02 | 用户注册 | 新用户注册账号 |
| F03 | 学生管理 | 学生信息增删改查 |
| F04 | 课程管理 | 课程信息增删改查 |
| F05 | 成绩管理 | 成绩录入、修改、删除 |
| F06 | 搜索查询 | 按姓名、班级、课程关键词搜索 |
| F07 | 成绩统计 | 各科平均分、及格率图表展示 |

### 1.3 非功能需求

- 界面简洁，操作方便
- 前后端分离通信，使用 Ajax 异步请求
- 数据持久化存储在 SQLite 数据库
- 代码结构清晰，便于维护和答辩讲解

### 1.4 可行性分析

**技术可行性：** 采用 Flask + SQLite + HTML/JavaScript，均为成熟免费技术，开发工具仅需 VS Code 和浏览器。

**经济可行性：** 无需购买服务器和数据库软件，本地即可运行演示。

**操作可行性：** 页面采用表格和表单布局，符合用户使用习惯。

### 3.4 关键流程

1. 用户打开 `login.html`，输入账号密码
2. 前端通过 `fetch` 调用 `/api/login`
3. 登录成功后跳转首页，sessionStorage 保存用户信息
4. 各功能页通过 `api.js` 封装的方法调用后端接口
5. 后端 `app.py` 接收请求，调用 `db.py` 操作数据库并返回 JSON

### 4.2 后端接口（app.py 节选）

```python
@app.route("/api/login", methods=["POST"])
def api_login():
    data = request.get_json() or {}
    user = db.login_user(data.get("username", "").strip(),
                         data.get("password", "").strip())
    if user:
        return ok(user)
    return fail("login failed")
```

### 4.3 前端 Ajax 封装（api.js 节选）

```javascript
function apiRequest(url, method, body) {
  var options = {
    method: method || "GET",
    headers: { "Content-Type": "application/json" }
  };
  if (body) options.body = JSON.stringify(body);
  return fetch(url, options).then(function (res) { return res.json(); });
}

var API = {
  login: function (username, password) {
    return apiRequest("/api/login", "POST", { username: username, password: password });
  },
  getGrades: function (keyword) {
    var url = "/api/grades";
    if (keyword) url += "?keyword=" + encodeURIComponent(keyword);
    return apiRequest(url);
  }
};
```

## 6 总结

本次实习完成了基于 Flask + SQLite + HTML/JavaScript 的学生成绩管理系统，实现了用户认证、学生/课程/成绩 CRUD、搜索查询和统计图表等功能。通过项目实践，掌握了 Web 前后端分离开发的基本流程，理解了 Ajax 异步通信和 SQLite 数据库操作。

**不足与改进：** 密码为明文存储，安全性不足；未实现成绩修改前端入口；后续可增加权限分级、数据导出和更完善的表单校验。
